# TJU Thesis Reviewer

适用于天津大学本科毕业设计（论文）的审查与整改场景。

## 触发场景

- 用户要求按天津大学本科毕业论文规范审稿
- 用户要求检查终稿、送审稿或答辩前版本
- 用户要求重点检查前置页、图表、摘要、参考文献、附录或语体

## 首先做什么

1. 读取仓库根的 `PROJECT.md`
2. 若存在 `MEMORY.md`，一并读取
3. 确认主文件、摘要文件、正文文件、参考文献文件和附录文件

## 审查顺序

1. 前置页顺序、页码体系、页眉页脚、目录层级
2. 标题层级、段落、摘要与关键词、中英文对应
3. 图、表、公式、附录编号、题注位置、三线表
4. 参考文献体例与文内引用一致性
5. 学术语体与送审表达

## 关键约束

- 图题在图下，表题在表上
- 图、表、公式按章编号
- 表格优先三线表
- 图表术语、单位与正文一致
- 禁止把本地路径、内部 PDF、脚本文件名写成正文论据
- 禁止“答辩中应”“读者应”“为了方便展示”这类答辩备忘口吻

## 输出格式

按以下结构输出：

1. 问题清单（按严重程度）
2. 已修改或建议修改的文件
3. 已完成项与未完成项
4. 剩余风险
5. 编译结果
6. 建议优先复核的位置
